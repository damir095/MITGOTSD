# B3 — распознавание дорожных знаков

Два этапа, две модели:

1. **Stage 1 — YOLOv8n** (1 класс «sign»): находит знаки в кадре. Обучен на
   combined GTSDB (Германия) + RTSD (Россия).
2. **Stage 2 — EfficientNet-B0** (48 классов): классифицирует каждый
   найденный знак (43 немецких GTSRB + 5 русских RTSD).

Подписи на кадрах — на русском.

## 1. Установка

Python **3.10–3.12**, NVIDIA GPU желательно (CPU тоже работает, но медленнее
в десятки раз). Драйверы NVIDIA — свежие, иначе `torch.cuda.is_available()`
будет `False` и всё пойдёт на CPU.

```powershell
# Создать виртуальное окружение
python -m venv .venv
.venv\Scripts\activate

# PyTorch (CUDA 12.1) — отдельной командой, под свою систему см.
# https://pytorch.org/get-started/locally/
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu121

# Остальные зависимости
pip install ultralytics timm pandas scikit-learn opencv-python pillow tqdm
```

> ⚠️ `requirements.txt` в репо имеет битые версии torch — **не используй**
> `pip install -r requirements.txt`. Команды выше — рабочая комбинация.

Проверка GPU:
```powershell
python -c "import torch; print('CUDA:', torch.cuda.is_available())"
```

## 2. Что лежит где

## 3. Запуск на видео

```powershell
python -m src.yolo_pipeline --cam C:\путь\к\видео.mp4 --save out.mp4 --no-show --yolo-imgsz 1280
```

`out.mp4` создаётся рядом, с зелёными боксами и русскими подписями.

## 4. Запуск с веб-камеры

```powershell
python -m src.yolo_pipeline --cam 0
```

Клавиши в окне: **`q`** — выход, **`s`** — снимок в `experiments/screenshots/`.

> Заметка: «поднеси знак к веб-камере в комнате» — это **вне распределения**
> обучения (телефонные/комнатные сцены). Реалистичные тесты — на дорожном видео.

## 5. Ключевые параметры

| Флаг | Дефолт | Что делает |
|---|---|---|
| `--yolo-imgsz` | 960 | Разрешение инференса YOLO. На 1080p видео ставь **1280** (мелкие/далёкие знаки лучше ловит). 1920 — макс., но медленнее. |
| `--yolo-conf` | 0.35 | Порог уверенности детектора. Ниже (0.25) → больше детектов, в т.ч. ложных. |
| `--clf-conf` | 0.5 | Порог классификатора: знак скрыт, пока top-1 не выше. Фильтрует мутные/нераспознанные. |
| `--persist` | 5 | Сглаживание во времени: бокс «доживает» N кадров. 0 — отключить. |
| `--save FILE` | — | Записывать аннотированное видео сюда. |
| `--no-show` | — | Без GUI-окна (нужен при `--save` в headless-режиме). |
| `--cam X` | 0 | Индекс камеры (`0`/`1`/…) или путь к видеофайлу. |
| `--yolo PATH` | `runs/detect/.../best.pt` | Свой детектор. |
| `--clf PATH` | `experiments/checkpoints/best.pt` | Свой классификатор. |

## 6. Замер качества (опционально)

Эти метрики имеют смысл только если у тебя есть **датасеты валидации**:
- **GTSDB val** (немецкие сцены) — `c:\mitgo\datasets\gtsdb_yolo\images\val` + `gtsdb\TrainIJCNN2013\TrainIJCNN2013\gt.txt`. Без них `evaluate_b3` не запустится.
- **RTSD val** (русские сцены) — `c:\mitgo\datasets\rstd\val_anno.json` + `rtsd-frames/`. Без них `evaluate_yolo_rtsd` не запустится.

```powershell
# Немецкая часть + классификатор + e2e на реальных боксах
python -m src.evaluate_b3

# Детектор на русских сценах — recall по 5 RU классам и в целом
python -m src.evaluate_yolo_rtsd
```

Текущие числа (для сверки):
- **`evaluate_b3`**: детектор recall **0.977**, классификатор-на-true **0.931**, end-to-end **0.915**.
- **`evaluate_yolo_rtsd`** (600 кадров): per-class recall 0.96–1.00, overall **0.955**.

## 9. Известные ограничения

- **Веб-камера в комнате / знак к экрану** — система обучена на дорожных
  сценах. Распознавание комнатных тестов сильно хуже, чем на видео с дороги.
- **Точность немецкого ~93%** на честной in-domain метрике (GTSDB val) —
  это цена добавления 5 русских классов. До-RU версия давала ~99%, но не
  знала русских знаков.
- **5 русских классов** только: пешеходный переход, искусственная неровность,
  парковка, остановка запрещена, стоянка запрещена. Других русских знаков
  модель не различает — относит к ближайшему по виду немецкому.
